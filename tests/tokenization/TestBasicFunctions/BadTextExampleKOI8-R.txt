<tbody><tr><td><h2><a href="https://opennet.ru/64709-kernel" class="nolink" onclick="return false;"><span id="r_title">
Набор подсказок для рецензирования изменений в ядре Linux и systemd при помощи AI
</span></a></h2>
<span style="font-size: 10pt;">02.02.2026&nbsp;13:17 (MSK)</span>
</td></tr>
</tbody></table>

<a name="link"></a><table class="ttxt2">
<tbody><tr><td class="chtext" bgcolor="E9EAD6" id="r_memo" style="padding-top: 15px;">
<p>Крис Мейсон (<a href="https://github.com/masoncl">Chris Mason</a>), создатель и мэйнтейнер файловой системы Btrfs, <a href="https://lore.kernel.org/lkml/b187e0c1-1df8-4529-bfe4-0a1d65221adc@meta.com/">опубликовал</a> проект <a href="https://github.com/masoncl/review-prompts">review-prompts</a>, содержащий коллекцию скриптов и подсказок для настройки процесса рецензирования изменений в <a href="https://github.com/masoncl/review-prompts/tree/main/kernel">ядре Linux</a> и системном менеджере <a href="https://github.com/masoncl/review-prompts/tree/main/systemd">systemd</a>,
 используя AI-ассистенты, такие как Сlaude Code. В состав включены файлы
 для уточнения особенностей работы различных подсистем и протоколов, 
позволяющие AI-ассистенту лучше понимать контекст при рецензировании, 
отладке и проверке изменений. 

</p><p>Также предоставляются шаблоны и списки проверок для выявления и классификации типовых ошибок (примеры для <a href="https://github.com/masoncl/review-prompts/blob/main/systemd/dbus.md">dbus</a> и <a href="https://github.com/masoncl/review-prompts/blob/main/kernel/networking.md">сетевой подсистемы</a>)  и <a href="https://github.com/masoncl/review-prompts/tree/main/kernel/agent">инструкции</a>
 по формированию через AI автоматизированных отчётов. После 
рецензирования патчей на выходе формируется файл  review-inline.txt, 
оформленный в виде заготовки email для отправки в список рассылки 
разработчиков ядра.

</p><p>Отмечается, что многие ложные срабатывания при анализе изменений 
через AI вызваны непониманием специфики кодовой базы и опубликованные 
подсказки нацелены на предоставление AI необходимой информации для 
принятия верных решений. В настоящее время уровень ложных срабатываний 
при использовании предложенного набора составляет примерно 10%.
